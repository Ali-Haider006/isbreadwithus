# Registration Form Fields - Final Specification (Option B)

## Form Fields List

1. **Name**
   - Type: Text input
   - Required: Yes
   - Validation: Min 2 characters, letters and spaces only
   - Placeholder: "Enter your full name"

2. **Phone Number**
   - Type: Text input
   - Required: Yes
   - Validation: Pakistan phone format (+92 or 03xx format)
   - Placeholder: "+92 300 1234567"

3. **Email**
   - Type: Email input
   - Required: Yes
   - Validation: Valid email format
   - Placeholder: "your.email@example.com"

4. **No-Show User Checkbox**
   - Type: Checkbox (single)
   - Required: No (optional)
   - Label: "I missed the previous meetup and want to share my previous payment"
   - Default: Unchecked
   - Behavior: When checked, shows field #5 and hides field #6

5. **Previous Payment Reference**
   - Type: Text input
   - Required: Only if checkbox (#4) is checked
   - Validation: Min 3 characters
   - Placeholder: "Enter your previous payment reference"
   - Visibility: Hidden by default, shown only when checkbox is checked
   - Behavior: Cleared when checkbox is unchecked

6. **Payment Screenshot**
   - Type: File upload (image)
   - Required: Only if checkbox (#4) is unchecked
   - Accepted formats: JPG, PNG, JPEG
   - Max size: 5MB
   - Features: Image preview after upload
   - Storage: Converted to base64 string
   - Visibility: Shown by default, hidden when checkbox is checked
   - Label: "Upload Payment Screenshot"

---

## Conditional Logic Summary

**Checkbox Unchecked (New User):**
- Previous Payment Reference: Hidden
- Payment Screenshot: Visible + Required

**Checkbox Checked (No-Show User):**
- Previous Payment Reference: Visible + Required
- Payment Screenshot: Hidden + Not Required
